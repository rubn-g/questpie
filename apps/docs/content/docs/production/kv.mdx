---
title: Key-Value Store
description: In-memory or custom-adapter key-value store for caching and session data.
---

QUESTPIE provides a KV store for caching, rate limiting, and ephemeral data.

## Configuration

### Default In-Memory

```ts title="questpie.config.ts"
export default runtimeConfig({
	kv: {
		defaultTtl: 3600,
	},
});
```

If `adapter` is omitted, QUESTPIE uses the in-memory adapter.

### Custom Adapter

```ts
kv: {
  adapter: myKvAdapter,
  defaultTtl: 3600,
}
```

## Usage

Access through the `kv` context:

```ts
handler: async ({ kv }) => {
	// Set
	await kv.set("key", "value", 3600);

	// Get
	const value = await kv.get("key");

	// Delete
	await kv.delete("key");
};
```
