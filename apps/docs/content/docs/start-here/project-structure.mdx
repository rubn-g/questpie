## Discovery Rules

Codegen discovers files based on **directory name** and **export pattern**:

| Directory | Export | Key derivation | Example |
|---|---|---|---|
| `collections/` | Default or named | Filename тЖТ camelCase | `blog-posts.ts` тЖТ `blogPosts` |
| `globals/` | Default or named | Filename тЖТ camelCase | `site-settings.ts` тЖТ `siteSettings` |
| `routes/` | Default | Filename тЖТ camelCase/slash path | `create-booking.ts` тЖТ `createBooking` |
| `jobs/` | Default | Filename тЖТ camelCase | `send-email.ts` тЖТ `sendEmail` |
| `routes/` (raw) | Default | Filename тЖТ slash path | `webhook.ts` тЖТ `webhook` |
| `services/` | Default | Filename тЖТ camelCase | `blog.ts` тЖТ `blog` |
| `blocks/` | Named exports | Export name | `export const hero` тЖТ `hero` |
| `emails/` | Default | Filename тЖТ camelCase | `appointment-confirmation.ts` тЖТ `appointmentConfirmation` |

### Single-file conventions

Some configs are single files instead of directories:

| File | Purpose |
|---|---|
| `questpie.config.ts` | Runtime config (DB, plugins, adapters) |
| `modules.ts` | Module dependencies array |
| `auth.ts` | Authentication config |
| `locale.ts` | Content locale config |
| `sidebar.ts` | Admin sidebar sections |
| `dashboard.ts` | Admin dashboard widgets |
| `branding.ts` | Admin branding (name, logo) |
| `admin-locale.ts` | Admin UI locale |

### Nested routes

Routes support nested directories for namespacing:

```text
routes/
тФЬтФАтФА booking/
тФВ   тФЬтФАтФА create.ts          тЖТ client.routes.booking.create()
тФВ   тФФтФАтФА cancel.ts          тЖТ client.routes.booking.cancel()
тФЬтФАтФА get-stats.ts           тЖТ client.routes.getStats()
```

## The `.generated/` Directory

Running `bunx questpie generate` creates `.generated/` with:

### `index.ts` тАФ App instance and types

```ts
// Auto-generated тАФ do not edit
import type { AppCollections, AppGlobals, AppRoutes, AppJobs } from "./module";

export type AppConfig = {
  collections: AppCollections;
  globals: AppGlobals;
  routes: AppRoutes;
  // ... auth, locales
};

export { app, createContext } from "./app";
export type { AppConfig, AppCollections, AppGlobals, AppRoutes, AppJobs };
```

### Module augmentation

The generated code augments `AppContext` so every handler gets typed DI:

```ts
declare global { namespace Questpie {
  interface AppContext {
    db: Database;
    email: MailerService<AppEmailTemplates>;
    queue: QueueClient<AppJobs>;
    collections: AppCollections;
    globals: AppGlobals;
    session: Session | null;
    // ... custom services
  }
}}
```

This is why `collections`, `queue`, `email` are available and typed in every handler тАФ codegen generates the augmentation from your file structure.

## The `#questpie` Import

Collection and global files use `#questpie` as the import specifier:

```ts
import { collection } from "#questpie";
```

This is a TypeScript path alias that resolves to the generated app instance. It ensures your builders have access to the correct types for field autocompletion and relation targets.

## By-Feature Layout

