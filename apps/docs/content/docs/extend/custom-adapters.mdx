QUESTPIE ships with adapters for Hono, Elysia, and Next.js. You can create your own for other frameworks.

## Adapter Pattern

An adapter converts framework-specific requests to standard `Request` objects and returns `Response` objects:

```ts
import { createFetchHandler } from "questpie";

export function createMyFrameworkAdapter(app, options) {
	const handler = createFetchHandler(app, options);

	return async (frameworkRequest) => {
		// Convert framework request to standard Request
		const request = toStandardRequest(frameworkRequest);

		// Handle with QUESTPIE
		const response = await handler(request);

		// Convert standard Response back to framework format
		return toFrameworkResponse(response);
	};
}
```

## Generic Fetch Handler

The simplest approach is using `createFetchHandler` directly:

```ts
import { createFetchHandler } from "questpie";
import { app } from "#questpie";

const handler = createFetchHandler(app, { basePath: "/api" });

// Use with any framework that supports standard Request/Response
const response = await handler(request);
```

This is exactly how TanStack Start integration works — no adapter package needed.
